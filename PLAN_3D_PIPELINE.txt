================================================================================
OVERVIEW
================================================================================
The current pipeline benchmarks 6 methods on 2D slices for 4 synaptic
structures. The next run retains ALL existing 2D methods and adds two new
architecture classes (2.5D and 3D), each tested with the SAME combination
optimization matrix (DA, fine-tuning variants, TTA, ensemble, threshold sweep).
Post-hoc 3D consistency is applied as an additional optimization layer on top
of all 2D predictions.

This produces a comprehensive benchmark: 3 architectures × 6 methods ×
combination optimizations × 4 structures.

================================================================================
PHYSICAL SCALE (CURRENT DATASET)
================================================================================
XY pixel size:       5.0 nm/px (TIFF metadata, confirmed from DM4 source)
                     Raw DM4 value: 4.681 nm/px (32.768 µm FOV / 7000 px)
                     ImageJ/TIFF rounded to 200 px/µm = 5.0 nm/px
Z section spacing:   ~54 nm (TrakEM2 layer thickness 10.79 units × 5.0 nm/px)
Z:XY anisotropy:     ~10.8:1 (54 nm z / 5 nm xy)
Image size:          7000 × 7000 px per montage (4×1 tiles, 15% overlap)
Imaging:             Zeiss SEM + DigiScan, 2 kV, serial ultra-thin sections

FULL METHOD MATRIX
================================================================================

ARCHITECTURE A: 2D U-Net (CURRENT — retained as-is)
─────────────────────────────────────────────────────
  Methods (per structure):
    1. Unadapted (pretrained SynapseNet baseline)
    2. Domain adapted (mean-teacher semi-supervised)
    3. Fine-tuned with Dice loss
    4. Fine-tuned with focal loss + foreground weighting
    5. DA + fine-tuned (DA then supervised fine-tuning)
    6. micro-SAM zero-shot (vit_b_em_organelles, AMG)
  Combination optimization:
    - Threshold sweep (0.05–0.90)
    - Test-time augmentation (7 geometric augmentations)
    - Model ensemble (max-voting across compatible models)
  Post-hoc (NEW):
    - 3D consistency enforcement on stacked 2D predictions (see below)

ARCHITECTURE B: 2.5D U-Net (NEW)
─────────────────────────────────
  Input: 3 adjacent slices as multi-channel input (N-1, N, N+1)
  Base model: SynapseNet 2D U-Net with in_channels=3
  Methods (same matrix as 2D):
    1. Unadapted (pretrained, replicate 1ch weights across 3 channels ÷ 3)
    2. Domain adapted (mean-teacher with 3-channel input)
    3. Fine-tuned with Dice loss
    4. Fine-tuned with focal loss + foreground weighting
    5. DA + fine-tuned
    6. micro-SAM zero-shot (feed middle slice; 2.5D context N/A)
  Combination optimization:
    - Threshold sweep
    - TTA (7 geometric augmentations applied identically to all 3 channels)
    - Model ensemble
  Notes:
    - Minimal architecture change (in_channels 1→3)
    - Same GPU memory / training time as 2D
    - Edge handling: replicate first/last slice at stack boundaries

ARCHITECTURE C: 3D AnisotropicUNet (NEW)
─────────────────────────────────────────
  Input: 3D volume patches (Z_depth × 256 × 256)
  Base model: SynapseNet pretrained 3D AnisotropicUNet
  Voxel size: (54.0, 5.0, 5.0) nm for anisotropic kernel selection
  Methods (same matrix as 2D):
    1. Unadapted (pretrained 3D baseline)
    2. Domain adapted (mean-teacher 3D with unlabeled stack volumes)
    3. Fine-tuned with Dice loss
    4. Fine-tuned with focal loss + foreground weighting
    5. DA + fine-tuned
    6. micro-SAM zero-shot (N/A — micro-SAM is 2D only; skip or report N/A)
  Combination optimization:
    - Threshold sweep
    - TTA (3D augmentations: flips in xy, rotations; restricted z-augmentation)
    - Model ensemble (if multiple 3D models available)
  Notes:
    - Requires >= 8–16 annotated serial sections for meaningful z-context
    - Higher GPU memory (RTX 3090 24 GB for training, batch_size 1–2)
    - AnisotropicUNet handles 10.8:1 z:xy anisotropy natively
    - 3D tiled inference with overlap in all dimensions

POST-HOC 3D CONSISTENCY (independent toggle on Architecture A outputs)
─────────────────────────────────────────────────────────────────────
  An additional optimization layer applied to 2D predictions after stacking
  across z. Every 2D method is evaluated BOTH with and without post-hoc 3D
  consistency, so the effect can be measured independently.

  Methods:
    a) 3D connected components:
       - Stack 2D masks → 3D connected components → filter by volume
    b) Instance tracking:
       - Link instances across adjacent slices by overlap
       - Merge fragmented detections, remove single-slice noise
    c) Z-smoothing:
       - Gaussian/median filter on probability maps along z → re-threshold
    d) 3D CRF (optional):
       - Conditional random field on 3D probability volume

  Evaluation:
    - Each 2D configuration reported twice: without 3D-CC and with 3D-CC
    - Measures how much 3D consistency alone improves 2D predictions
    - Provides a fair comparison point: does 2D + 3D-CC match native 3D?

EVALUATION MATRIX

All methods evaluated with:
  Per-slice (2D) metrics:
    - Dice coefficient, IoU, precision, recall
    - Instance F1, instance count accuracy
  Volumetric (3D) metrics (NEW):
    - Volumetric Dice (3D binary overlap)
    - 3D instance segmentation metrics (track-level F1)
    - Cross-slice consistency (% instances tracked across z)
    - Surface smoothness (mesh roughness metric)
  Physical-unit morphometrics:
    - Areas in nm², volumes in nm³
    - Lengths/perimeters in nm
    - 3D surface area in nm²

================================================================================
PER-STRUCTURE CONSIDERATIONS
================================================================================
Mitochondria:
  - Span many sections → biggest beneficiary of 3D/2.5D context
  - 3D instance segmentation enables per-organelle volume measurement
  - Current 2D ensemble (3 models, max-vote) is strong baseline to beat
  - Expect 3D AnisotropicUNet to outperform 2D ensemble

Presynaptic membrane:
  - Continuous structure spanning entire stack
  - 3D model enforces topological continuity (single connected surface)
  - Post-hoc 3D connected components alone may close per-slice gaps
  - 3D surface area becomes a meaningful morphometric

Synaptic ribbon:
  - Small, spans 1–3 sections
  - 3D context helps boundary precision more than detection
  - 3D volume measurement useful for ribbon size quantification
  - Ribbon-proximity masking for vesicles extends to 3D (spherical mask)

Synaptic vesicles:
  - Smallest structures (~30–100 px, span 1–2 sections)
  - 3D context helps distinguish overlapping vesicles
  - Ribbon-proximity prior remains valuable (spherical distance mask in 3D)
  - May need higher-resolution imaging for true individual detection

DATASET REQUIREMENTS FOR NEXT RUN
================================================================================
Minimum:
  - >= 8 serial sections per annotated region (for 3D U-Net training)
  - >= 2 annotated regions (train/test split)
  - All 4 structures annotated on every section
  - Consistent imaging conditions across sections

Ideal:
  - 20–50 serial sections per region
  - 3–5 annotated regions from different tissue blocks
  - Multiple tissue types or retinal regions for generalizability
  - Section thickness measured/recorded (ultramicrotome settings or EELS)

Metadata to record:
  - Exact section thickness (nm)
  - XY pixel size (nm/px) from calibrated magnification
  - Imaging voltage, detector, dwell time
  - Tissue type, species, preparation method
